# salon-seat-scheduler

## V0.9.0 基礎與自訂項目版

V0.8.3 保留原有預約管理與 Firestore 即時同步。單做美甲可直接選擇「美甲」項目與 13 安妮蒂芬妮藍；美髮加美甲時，第二服務會依日期、開始時間與時長，以蒂芬妮藍細長條疊加在完整主預約卡上。跨日第二服務只顯示精簡排程提示。

預約高度依店內紙本格線計算，結束時間所在的半小時格也會完整覆蓋。主預約與第二服務各自依開始時間、時長計算位置與高度；重疊判定、自動排位及空格檢查也使用相同規則。

預約表支援手機雙指縮放、桌機 Ctrl／Command 加滾輪，以及放大、縮小、100% 與「適合畫面」控制。縮放只改變整張表的顯示比例，不改變預約與時間格的座標，縮放後仍可捲動及操作預約。

系統內建前一天、今天、明天與 Firebase 重整按鈕，不依賴瀏覽器工具列。手機和平板可在表格水平邊界左右滑切換日期，或在表格頂端下拉重新同步；手勢設有方向、距離與邊界判定，以保留表格捲動、預約操作及雙指縮放。網站也提供 Web App manifest 與 service worker，可在支援的平台安裝成獨立視窗使用。

高風險功能集中於「設定 → 危險操作」，包含各層級刪除、批次刪除及 JSON 匯入／匯出。每次操作都經過警告、管理密碼、第二次確認與最後確認；測試密碼為 `2255`，連續錯誤 5 次會鎖定 5 分鐘。刪除資料會進入 Firestore 垃圾桶保留 30 天並可復原，所有結果均寫入操作紀錄。此測試密碼位於前端程式，不應視為正式環境的安全驗證；正式上線應改用具管理員權限的 Firebase 帳號與後端驗證。

第一、第二階段先建立後續營業時間功能所需的 `startTime` 相容欄位，將「助理」全面改名為「技術師」，並把日期、導覽、重整與縮放控制列固定整合在表格上方。消費項目可在預約表單自行新增並透過 Firestore 保留；備註可選擇是否顯示在預約方塊，顯示時最多兩行並以省略方式避免破版。新功能從 `catalog.js` 與 `styles.css` 開始拆分，避免繼續集中於 `index.html`。

### Firebase 設定

1. 在 Firebase Console 建立或選擇專案，新增 Web App，啟用 Firestore Database 與 Authentication 的匿名登入。
2. 把 Web App SDK 設定填入 `firebase-config.js`。
3. 將 `firestore.rules` 部署到該 Firebase 專案。
4. 透過 HTTPS 網站開啟 `index.html`。多台裝置只要使用相同 Firebase 專案，新增、修改、刪除與拖曳就會即時同步。

> 匿名登入不會改變原有操作方式；Firestore 規則會拒絕未取得 Firebase 身分的存取。若需限制只有店內裝置可用，建議後續改用店員帳號登入。
